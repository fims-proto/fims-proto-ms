# FIMS System Overview

## What Is FIMS?

FIMS (Financial Information Management System / 财务信息管理系统) is a standalone accounting SaaS product for Chinese small and medium-sized enterprises (SMEs). Each customer receives an isolated deployment. The system currently implements the **Small Business Accounting Standards (小企业会计准则)**, with support for additional Chinese accounting standards on the roadmap.

## Deployment Model

FIMS is delivered as an out-of-the-box SaaS product — each customer gets their own isolated instance. It is **not** a shared-tenant platform. The exact hosting mechanism (self-hosted, managed cloud, etc.) is deferred.

## Core Concept: Set of Books (账套)

A **Set of Books (SoB / 账套)** is the top-level accounting entity. One FIMS deployment supports multiple SoBs, which allows a business to:

- Manage books for multiple subsidiary companies or legal entities, or
- (For accounting firms) manage books for multiple client companies.

Each SoB is fully independent with its own chart of accounts, accounting periods, journals, and ledger balances.

## Modules and Functions

### 1. Set of Books Management (账套管理)

Create and configure accounting entities. Each SoB is configured with:
- Name and description
- Base currency
- Starting fiscal year and month
- Chart of accounts code length structure (defines how many levels the account hierarchy has)

### 2. General Ledger (总账)

The core accounting module. Consists of four sub-areas:

#### Chart of Accounts (科目)

A hierarchical chart of accounts following Chinese accounting standards. Accounts are classified as:

| Class       | Chinese | Description                    |
| ----------- | ------- | ------------------------------ |
| Assets      | 资产    | What the business owns         |
| Liabilities | 负债    | What the business owes         |
| Equities    | 权益    | Owners' stake                  |
| Costs       | 成本    | Production and operating costs |
| P&L         | 损益    | Revenue and expense accounts   |
| Commons     | 共同    | Shared-use accounts            |

Each account has a balance direction (debit-normal or credit-normal). Leaf-level accounts are the only ones that can be directly posted to.

#### Journal Entries (凭证)

The daily accounting record. Each journal entry has a header (period, date, document number, attachments) and one or more journal lines (account + signed amount).

**Journal lifecycle (four-eye workflow):**

```
Create → Review (复核) → Audit (审核) → Post (登账)
```

- Each role must be a different person (segregation of duties).
- Only posted journals affect ledger balances.
- Review and audit can be cancelled to revert the journal to an earlier state.
- User roles (accountant, auditor, CFO, etc.) are on the roadmap; not yet enforced by the system.

**System journals** are auto-generated by the system at period close:
- **Monthly Closing Journal (月末结账凭证)**: Clears P&L account balances and transfers the net result to Current Year Profit (本年利润, account 003103).
- **Year-End Closing Journal (年末结账凭证)**: Transfers Current Year Profit to Retained Earnings (未分配利润, account 003104000002) at the end of period 12.

System journals follow the same four-eye workflow before posting, and can be deleted (reversing their ledger impact) if needed.

#### Accounting Periods (账期)

Monthly accounting periods generated per fiscal year. A period can be **open** (accepts new journal postings) or **closed** (sealed, no further postings).

**Period closing (结账)** validates:
1. All journals in the period are posted.
2. P&L account balances are zero (cleared by Monthly Closing Journal).
3. Trial balance passes (sum of all signed amounts = 0).
4. For period 12: Current Year Profit account is zero (cleared by Year-End Closing Journal).

**Batch closing (连续结账)**: Closes multiple consecutive periods in one atomic operation (up to 12 periods).

#### Ledger Balances (账本)

Tracks account balances per period. For each account and period, the system maintains:
- Opening balance
- Period debit / period credit movement
- Ending balance

Balances are updated automatically when a journal is posted. Ledger data feeds directly into report generation.

---

### 3. Financial Reports (报表)

Generates standard financial statements for a given period. Two report types are supported:

| Report           | Chinese    | Amount Types Available                               |
| ---------------- | ---------- | ---------------------------------------------------- |
| Balance Sheet    | 资产负债表 | Year opening balance, Period ending balance          |
| Income Statement | 利润表     | Period amount, Year-to-date amount, Last year amount |

Reports are built from a configurable section/item structure. **Standard templates are pre-shipped** with each SoB and can be customised. A report is generated (or re-generated) against a specific accounting period, pulling data from ledger balances.

---

### 4. Dimensions (维度)

Enables multi-dimensional financial analysis. Users define:

- **Dimension Categories (维度分类)**: A classification axis, e.g. "Department", "Project", "Cost Center".
- **Dimension Options (维度选项)**: Values within a category, e.g. "Engineering", "Marketing" under "Department".

Accounts can be bound to one or more dimension categories, making them **required** when recording a journal line against that account. This allows ledger balances to be broken down by dimension (e.g., expenses by department) without creating a separate account for each combination.

---

### 5. Numbering (自动编号)

Auto-generates sequential document numbers for journal entries. Configurations define the numbering pattern and maintain an auto-increment counter per accounting period. This ensures every journal has a unique, human-readable identifier (e.g., `记 1 号`).

---

### 6. User Management (用户管理)

Manages user profiles. Authentication is handled externally (Ory Kratos). Role-based access control (accountant, reviewer, auditor, CFO, etc.) is on the roadmap and not yet enforced by the system.

## Key Business Workflows

### Initial Setup
1. Create a Set of Books (SoB) with fiscal year and account code structure.
2. The system seeds a standard Chart of Accounts based on 小企业会计准则.
3. Configure dimension categories for the accounts that need them.
4. Configure numbering rules.
5. Initialize ledger opening balances (for companies migrating from another system).

### Daily Accounting
1. Accountant creates a journal entry (journal lines with account + amount + optional dimensions).
2. Reviewer reviews the journal.
3. Auditor audits the journal.
4. Poster posts the journal — ledger balances are updated.

### Month-End Close
1. Verify all journals for the period are posted.
2. Generate and post the Monthly Closing Journal (clears P&L to Current Year Profit).
3. Close the period — the system validates trial balance and P&L clearance.

### Year-End Close (Period 12 only)
1. Complete month-end close steps.
2. Generate and post the Year-End Closing Journal (transfers Current Year Profit to Retained Earnings).
3. Close period 12.

### Reporting
1. Select a report template (Balance Sheet or Income Statement).
2. Choose a target period.
3. Generate the report — the system reads ledger balances and evaluates the report formula structure.
4. Re-generate if ledger data changes after initial generation.

## Accounting Standard

FIMS currently implements **小企业会计准则 (Small Business Accounting Standards)**. The standard chart of accounts is pre-loaded as seed data (`dataload/xqykjzz/`). Support for additional Chinese accounting standards (e.g., 企业会计准则) is on the roadmap.
